# Проект по категоризации заявок (categorisation_TH)

Данный проект представляет собой автоматизированную систему для классификации заявок в техподдержку с использованием NLP и машинного обучения. Система использует гибридный пайплайн, поддерживающий как классический подход TF-IDF, так и современные эмбеддинги на базе Трансформеров.

## Структура проекта

- `train_thesis_model.py`: Скрипт для обучения моделей-кандидатов (Logistic Regression, SVC, Random Forest) с использованием 5-кратной кросс-валидации и выбора лучшей модели.
- `process_request.py`: Скрипт для инференса (одиночного или пакетного) в "продуктовой" среде с использованием сохраненных весов.
- `thesis_pipeline.py`: Основная логика пайплайна, содержащая класс `TransformerEmbeddingExtractor` и функции предобработки.
- `service/`: Микросервис на базе FastAPI для работы через API.
- `requirements_for_comparison.txt`: Полная среда для исследований, обучения и сравнения моделей (включает SHAP и Matplotlib).
- `requirements_for_use.txt`: Минимальная среда для работы инференса.
- `service/requirements_for_FastAPI.txt`: Среда для работы FastAPI сервиса.

## Быстрый старт

### Установка зависимостей

Выберите файл зависимостей в зависимости от ваших задач:

```bash
# Для исследований и обучения
pip install -r requirements_for_comparison.txt

# Только для работы готовой модели (инференс)
pip install -r requirements_for_use.txt
```

### Обучение

Чтобы обучить модели и сохранить лучшую из них:
```bash
python train_thesis_model.py --source data.csv --model best_model.pkl
```

### Обработка заявок (Инференс)

Чтобы обработать запрос на классификацию:
```bash
python process_request.py sample_input.json output_results.json --model best_model.pkl
```

### Запуск API сервиса

```bash
cd service
pip install -r requirements_for_FastAPI.txt
(python service/main.py)
uvicorn main:app --reload
(python service/test_api.py чтобы запустить тестовый запрос к микросервису)
```

## Ключевые особенности

- **Гибридный пайплайн**: Возможность переключения между TF-IDF и эмбеддингами Трансформеров.
- **Маппинг SLA**: Автоматическое определение срока ответа в зависимости от категории.
- **Назначение исполнителя**: Интеграция с `roles.json` для предложения ответственного сотрудника.
- **FastAPI Микросервис**: Готовый к работе асинхронный API для интеграции с другими системами.
